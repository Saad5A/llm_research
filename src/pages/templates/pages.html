What is a transformer ŌĆ?

Every model in this guide is built on the transformer architecture ŌĆ? introduced by Google in the 2017 paper "Attention Is All You Need." Transformers process all words in parallel, using self-attention to understand relationships between any two tokens regardless of distance.

Phase 03 ┬Ę Training objectives

How each model learns

The architecture tells you the shape of the model. The training objective tells you what it actually learns to do ŌĆ? and why three models built on the same transformer foundation behave so differently.

03.1

BERT ŌĆ? Masked Language Modelling

BERT ┬Ę Masked LM + Next Sentence Prediction

Fill in the blank,
predict the pair
01Masking: 15% of input tokens are randomly replaced with [MASK]. The model must predict the original token using only surrounding context.
02Bidirectional context: Because BERT sees the whole sentence at once, both left and right context inform every prediction ŌĆ? unlike earlier left-to-right models.
03Next Sentence Prediction (NSP): Given two sentences, BERT also learns to predict whether sentence B naturally follows sentence A ŌĆ? building paragraph-level understanding.
04No labels needed: Both tasks are self-supervised ŌĆ? the labels come directly from the text itself, enabling training on billions of tokens without any human annotation.

03.2

Claude ŌĆ? RLHF + Constitutional AI

Claude ┬Ę Reinforcement Learning from Human Feedback + Constitutional AI

Learn from humans,
then from principles
1
Pretraining
Train on a massive corpus of text to build general language understanding and generation capability ŌĆ? the same as any large language model.
Ōå?
2
Supervised Fine-Tuning (SFT)
Fine-tune on human-written examples of ideal responses ŌĆ? helpful, harmless, honest conversations across a wide variety of tasks.
Ōå?
3
Reward Model Training
Human raters compare pairs of model outputs and rank them. These preferences train a separate reward model that scores response quality.
Ōå?
4
Reinforcement Learning (PPO)
Use the reward model to fine-tune the LLM via Proximal Policy Optimisation ŌĆ? maximising the reward signal while staying close to the SFT policy.
Ōå?
5
Constitutional AI (Anthropic-specific)
The model critiques and revises its own outputs against a written set of principles ŌĆ? reducing reliance on human feedback for safety, and making the values transparent and auditable.

Sample Constitutional AI principles

Choose the response that is least likely to contain harmful or unethical content.
Choose the response that is most honest and acknowledges uncertainty where it exists.
Choose the response that is most helpful to the human, given that it is also safe and ethical.
Choose the response that a thoughtful senior Anthropic employee would consider most appropriate.
03.3

LLaMA ŌĆ? Causal Language Modelling at scale

LLaMA ┬Ę Next-token prediction ┬Ę 15 trillion tokens

Predict what comes next,
at massive scale
01Causal LM objective: Given a sequence of tokens, predict the next one. Simple, but applied over 15 trillion tokens of diverse text ŌĆ? books, code, web, scientific papers.
02Efficiency first: LLaMA was designed to be maximally capable within a given compute budget ŌĆ? smaller models trained on more data, informed by the Chinchilla scaling laws.
03Open weights: Meta releases the trained weights publicly. The community fine-tunes them with instruction datasets (Alpaca, ShareGPT) to create instruction-following variants.
04No alignment by default: Base LLaMA is a raw language model. Safety and alignment are the responsibility of whoever fine-tunes it ŌĆ? a deliberate choice enabling research freedom.

BERT sentiment classifier ŌĆ? Fine-tuned on SST-2 (movie reviews). The pipeline abstraction handles tokenisation, inference, and decoding automatically. Best for: product reviews, social media monitoring, support ticket triage.

# pip install transformers torch
from transformers import pipeline

# Load a BERT model fine-tuned on SST-2 sentiment

    "sentiment-analysis",
    model="distilbert-base-uncased-finetuned-sst-2-english",
    device=-1  # -1 = CPU; use 0 for GPU

texts = [

    "I couldn't get the model to run at all.",
    "It's an interesting paper but the results are mixed.",

Claude via Anthropic SDK ŌĆ? Structured reasoning with JSON output. Claude's long context and instruction-following make it ideal for analysis tasks that require nuanced judgment, not just classification. The system prompt controls persona and output format.

# pip install anthropic
import anthropic, json

client = anthropic.Anthropic()  # reads ANTHROPIC_API_KEY from env

paper_abstract = """
    We introduce a novel attention mechanism that reduces
    quadratic complexity to linear while maintaining
    performance on long-document benchmarks...
"""

message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system="""You are a research assistant. Analyse the given
abstract and return ONLY valid JSON with these keys:
- topic (str): the main research topic
- contribution (str): the key contribution in one sentence
- models_mentioned (list[str]): any ML models or methods named
- confidence (float 0-1): how confident you are in your analysis""",
    messages=[
        {"role": "user", "content": paper_abstract}
    ]
)

result = json.loads(message.content[0].text)
print(json.dumps(result, indent=2))

LLaMA 3 running locally ŌĆ? 4-bit quantisation via bitsandbytes lets you run an 8B model on a single 8GB GPU (or ~12GB RAM on CPU). This is the key advantage of open weights: no API calls, no data leaving your machine, full customisation.

# pip install transformers torch accelerate bitsandbytes
import torch
from transformers import (
    AutoTokenizer,
    AutoModelForCausalLM,
    BitsAndBytesConfig,
    pipeline,
)

# 4-bit quantisation ŌĆ? runs on ~8GB VRAM
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
)

model_id = "meta-llama/Meta-Llama-3-8B-Instruct"

tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    quantization_config=bnb_config,
    device_map="auto",
)

pipe = pipeline(
    "text-generation",
    model=model,
    tokenizer=tokenizer,
    max_new_tokens=256,
)

prompt = """<|begin_of_text|><|start_header_id|>user<|end_header_id|>
Explain the difference between BERT and LLaMA in two sentences.
<|eot_id|><|start_header_id|>assistant<|end_header_id|>
"""

output = pipe(prompt)[0]["generated_text"]
print(output.split("assistant")[-1].strip())
BERT is an encoder-only model trained with masked language modelling to understand and classify text, while LLaMA is a decoder-only model trained with causal language modelling to generate coherent text. The key distinction is that BERT excels at tasks requiring deep text understanding, whereas LLaMA is designed for text generation and can be fine-tuned for a wide range of generative applications.
